1 Introduction
Financial markets exhibit complex and random behavior, making accurate modeling of
asset prices a challenging task. Traditional models such as the Black-Scholes Model,
and simple statistical models, often fail to capture important empirical features such as
time-varying volatility and extreme market events.
The objective of this project is to:
 Model asset price dynamics using Geometric Brownian Motion (GBM) and the
Heston Stochastic Volatility model.
 Estimate financial risk using Value-at-Risk (VaR).
 Compare simulation outputs with real market data.
 Improve model accuracy using an optimization-based calibration approach (an AI
component).

5.2 Value-at-Risk (VaR) Explained
Financial Meaning: Value-at-Risk answers a fundamental question for any investor or
bank: “Under normal market conditions, what is the maximum amount of money I can
expect to lose over a given time period, with a certain level of confidence?”
If an investment portfolio has a 1-year 95% VaR of -15%, it means we are 95% confident
that the portfolio will not lose more than 15% of its value over the next year. Conversely,
there is a 5% probability that the portfolio will experience a loss worse than 15%.
3

Mathematical Definition and Computation:
Mathematically, for a given confidence level 1 − α (where α ∈ (0, 1) is the significance
level), the VaR is defined as the threshold value such that the probability of the loss
exceeding this threshold is exactly α:
P (R ≤ −VaRα) = α
Since we are using Monte Carlo simulations, we compute this empirically using order
statistics. Let N be the total number of simulated paths ( N = 10 , 000). We calculate
the return for each path, yielding a set of returns {R1, R2, . . . , RN }. We then sort these
returns from the lowest (most severe loss) to the highest (greatest gain):
R(1) ≤ R(2) ≤ · · · ≤ R(N)
The VaR at confidence level 1 − α is simply the absolute value of the return at the
α-quantile of this sorted array:
VaRα = −R([N ×α])
Applying this to our simulation:
 For the 95% VaR ( α = 0 .05): N × α = 10 , 000 × 0.05 = 500. We locate the
500th worst return in our sorted list. VaR 95% = −R(500).
 For the 99% VaR ( α = 0 .01): N × α = 10 , 000 × 0.01 = 100. We locate the
100th worst return in our sorted list. VaR 99% = −R(100).

Because the variance vt is hidden in real life, we must create a proxy (such as a 30-

calibration.
 θ (Long-term mean): The historical average of the variance proxy.

 v0 (Initial variance): The most recent value of the variance proxy.
 ξ (Vol of vol): The standard deviation of the daily changes in the variance proxy.
 ρ (Correlation): The statistical correlation between the daily stock returns and
the daily changes in the variance proxy.
 κ (Mean reversion speed): Found by running a linear regression where the
dependent variable (Y ) is the daily change in variance, and the independent variable
(X) is the distance from the mean.

8 Calibration (AI Component)
Because the historical proxy method is imperfect, we use an optimization algorithm to
fine-tune the Heston parameters.
8.1 Objective and Seed Values
Minimize the error over the parameter space ( κ, θ, ξ, ρ, v0). We will feed the historical
estimates calculated in Section 6.2 into the optimizer as the starting “seed” values to
ensure the algorithm converges efficiently.
8.2 Loss Function
We define a Loss Function (L) that calculates the penalty for how far off our simulation
is from the real market data.
L = w1(meansim − meanreal)2 + w2(variancesim − variancereal)2 + w3(VaRsim − VaRreal)2
8.3 Interpretation
The AI algorithm will start at our historical guesses, run the simulation, check the loss
L, and intelligently adjust the parameters to minimize the difference between simulated
outputs and reality.

11 Conclusion
This project combines stochastic modeling, statistical estimation, and algorithmic opti-
mization to improve financial risk estimation. While GBM provides a baseline, the Heston
model captures realistic market dynamics. By taking historical statistical estimates and
refining them through an AI-based calibration loop, this workflow forces the simulated
distributions to closely align with real observed market crashes and heavy tails. Accu-
rate financial modeling requires both an appropriate stochastic structure and effective
parameter calibration, which this project successfully demonstrates.
